# 設計仕様：01-superpowers.md 教材リストラクチャリング

**日付**: 2026-06-20  
**対象ファイル**: `docs/03-frameworks/01-superpowers.md`  
**目的**: 初心者・中級者向けの価値訴求を強化し、読者が「自分のための記事だ」と感じながら読み進められる構成に再設計する

---

## 背景と課題

### 読者プロファイル

- Claude Code や GitHub Copilot を使い始めているが「思い通りに動かない」「雑に動く」と感じている初心者〜中級者
- エージェントが「考えずに動く」という問題を体感しているが、解決策を知らない状態でこのページに到達する

### 現状の問題

現行ドキュメントは「概念の説明 → インストール手順 → スキル一覧」という**論理的な順序**で書かれている。しかしこの順序だと、読者は「自分の問題への答え」にたどり着く前に用語・手順の羅列に直面する。結果として「Superpowers が自分に何をしてくれるのか」が最後まで掴みにくい。

### 改善の方針

**アプローチ B：物語構造への全面再構成**を採用する。

- 情報の削除はゼロ（順番の組み替えと冒頭の新規追加のみ）
- 「共感 → 原因 → 解決策 → 動作確認 → 使い方」という読者の心理フローに沿った順序に変更
- 大胆な再構成を許可（既存セクションの制約なし）

---

## 新しいセクション構成

### ① 共感フック（新規追加）

**目的**: 読者の「あるある体験」を具体的なシーンで描写し、「これは自分の話だ」と感じさせる。

**内容**:
- 「仕様を聞かずに実装が始まった」シーン
- 「完了と言ったのに動かなかった」シーン  
- 「バグを直したら別の場所が壊れた」シーン
- 締め：「これはエージェントが賢くないのではなく、段取りを知らないから」

**執筆スタイル**: 二人称（「あなたは〜したことがありませんか？」）。セリフと状況描写を交えた具体的な場面。

---

### ② なぜそうなるのか（新規追加）

**目的**: 問題の根本原因（エージェントの「即コード」習慣）を簡潔に説明し、解決策への橋渡しをする。

**内容**:
- 人間のエンジニアが自然にやる5ステップ（要件確認→設計→計画→実装→確認）
- エージェントの標準動作（指示を受けたら即コード、②〜③がすっ飛ぶ）
- 「これは設計ではなくデフォルトの習慣の問題」という結論
- Superpowers への誘導

---

### ③ Superpowers とは（現行「概要」を書き換え）

**目的**: Superpowers の定義と価値を、①②の流れを受けた形で簡潔に説明する。

**内容**:
- 定義：エージェントに開発方法論を注入するプラグイン
- 開発者・リポジトリ情報
- Before/After 比較表（標準エージェント vs Superpowers 導入後）
- 仕組み：スキルという設計図
- 用語説明テーブル（現行の用語セクションを統合）

**変更点**: 現行の「概要」セクションを書き換え、現行の「用語の説明」セクションを③の末尾に統合する。

---

### ④ 導入後の変化 Before/After（現行「動作イメージ」を移動・拡充）

**目的**: 同じ依頼でエージェントの振る舞いがどう変わるかを具体的に示す。

**内容**:
- Before（Superpowers なし）：仕様確認なし、即実装、検証なしで「完了」
- After（Superpowers あり）：要件確認→設計選択→タスク分割→実装→テスト確認

**変更点**: 現行の「実際の使い方 → 動作イメージ」部分を独立したセクションとして前方に移動し、Before 側のシーンを新規追加する。

---

### ⑤ 基本ワークフロー（移動のみ）

**目的**: Before/After を見た読者が「実際はどういう流れで動くのか」を確認できる位置に配置する。

**変更点**: 現行の「基本ワークフロー」セクションをそのまま移動。内容変更なし。

---

### ⑥ インストールと使い方（移動のみ）

**目的**: 「使いたい」と思った読者が必要な手順を参照できるよう、価値訴求の後に配置する。

**変更点**: 現行の「対応エージェントとインストール方法」「Claude Code での使い方」「VS Code + GitHub Copilot での使い方」「インストールの確認方法」をまとめて移動。内容変更なし。

---

### ⑦ スキル一覧（移動＋一言追加）

**目的**: 詳細な参照情報として末尾に配置し、最初から全部覚えなくてよいことを明示する。

**変更点**:
- 見出しを「スキル一覧」→「スキル一覧（リファレンス）」に変更
- 冒頭に一文追加：「必要なときに参照してください。最初から全部覚える必要はありません。」
- 内容変更なし

---

### ⑧ 哲学・比較・位置づけ・次のステップ（移動のみ）

**目的**: 深く知りたい読者向けの補足情報として末尾に集約する。

**変更点**: 現行の「哲学」「自動起動の仕組み」「GitHub Copilot Skills との関係」「このチュートリアルでの位置づけ」「次のステップ」をそのまま末尾に配置。内容変更なし。

## 成功基準

- 読者が冒頭3スクリーン以内に「これは自分の問題への答えだ」と感じられる
- 既存の情報（インストール手順・スキル一覧・比較表）がすべて保持されている
- 「用語を先に覚えてから読む」ではなく「読みながら自然に理解できる」流れになっている

---

## 実装しないもの

- 情報の削除・省略
- 新しいセクション（⑤〜⑧以外）の追加
- 他のドキュメントファイルへの変更
- 画像・図表の新規作成
