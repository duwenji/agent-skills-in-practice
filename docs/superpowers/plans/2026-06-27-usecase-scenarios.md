# ユースケースシナリオ追加 実装計画

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 各章の冒頭に「## こんな状況に刺さる」セクションを追加し、上級者が自分のプロジェクトとの接点を冒頭で見つけられるようにする。

**Architecture:** Tier 1（実践スキル章 ×5）→ Tier 2（スキル作成・フレームワーク章 ×3）の順に、各ファイルの冒頭概要セクションの直後にシナリオブロックを挿入する。各タスクは独立しており、並行実行可能。

**Tech Stack:** Markdown 編集のみ。コード変更なし。

## Global Constraints

- シナリオは必ず 2〜3 個。4 個以上は書かない
- 各シナリオは「**[職種・役割]として**、[状況と困りごと]なとき」の形式で書く
- 「〜な人に向いています」形式は使わない（「〜なときに使う」が正しい形式）
- 挿入位置: `## このスキルについて`（または `## 概要`）セクションの直後
- 既存のセクション（`## なぜこのスキルが必要か` 等）は移動・削除しない
- 完了基準は spec ファイル `docs/superpowers/specs/2026-06-27-usecase-scenarios-design.md` を参照
